**Date**: 2026-04-20

## 1. System Health & Connectivity

### TC-01: Embed Service Health
- **Endpoint**: `GET /health` (Port 8000)
- **Input**: None
- **Expected**: HTTP 200, `{"status": "healthy", "service": "bgem3-mlx", ...}`
- **Purpose**: Verify the embedding service is up and MLX model is loaded.

### TC-02: Rerank Service Health
- **Endpoint**: `GET /health` (Port 8002)
- **Input**: None
- **Expected**: HTTP 200, `{"status": "healthy", "service": "reranker-mlx", ...}`
- **Purpose**: Verify the reranker service is up and MLX model is loaded.

## 2. Embedding Generation

### TC-03: Dense Embedding (Single)
- **Endpoint**: `POST /embed`
- **Input**: `["Hello world"]`
- **Auth**: Bearer token required
- **Expected**: HTTP 200, one 1024-dim vector.
- **Purpose**: Verify base dense embedding logic.

### TC-04: Dense Embedding (Batch)
- **Endpoint**: `POST /embed`
- **Input**: List of 32 texts.
- **Expected**: HTTP 200, 32 vectors of 1024-dim.
- **Purpose**: Verify batch processing efficiency.

### TC-05: Hybrid Embedding
- **Endpoint**: `POST /embed/hybrid`
- **Input**: `["MLX is powerful"]`
- **Expected**: HTTP 200, contains both `dense_embeddings` and `sparse_embeddings` (lexical weights).
- **Purpose**: Verify sparse weight extraction from XLM-R hidden states.

## 3. Reranking

### TC-06: Basic Reranking
- **Endpoint**: `POST /rerank`
- **Input**: `{"query": "MLX", "passages": ["MLX is for Apple", "Windows is for Intel"]}`
- **Expected**: HTTP 200, "MLX is for Apple" should have a significantly higher score.
- **Purpose**: Verify cross-encoder scoring logic.

## 4. Security & Validation

### TC-07: Authentication Security
- **Endpoint**: `POST /embed`
- **Input**: Any valid text list.
- **Auth**: Missing or invalid token.
- **Expected**: HTTP 401 Unauthorized.
- **Purpose**: Verify endpoint protection.

### TC-08: Batch Limit Enforcement
- **Endpoint**: `POST /embed`
- **Input**: List of 33+ texts.
- **Expected**: HTTP 429 Too Many Requests (per original service constraints).
- **Purpose**: Verify resource protection.

### TC-09: Empty Input Handling
- **Endpoint**: `POST /embed`
- **Input**: `[]`
- **Expected**: HTTP 422 Unprocessable Entity.
- **Purpose**: Verify robustness.
